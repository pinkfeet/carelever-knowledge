# KINNECT — Assessments & Components bulk load

Operational guide for bulk-creating KINNECT's **assessment catalog** (assessments,
components, variations, and their links) in `carelever_assessment` from four CSV
templates filled in by the KINNECT team.

**Scripts + templates:** [`kinnect-assessments-import/`](kinnect-assessments-import/)

- [`export.rb`](kinnect-assessments-import/export.rb) — dump the current staging catalog to the five CSVs (round-trips with the importer)
- [`build_workbook.rb`](kinnect-assessments-import/build_workbook.rb) — merge the CSVs into one `.xlsx` with a Legend sheet (for sharing with KINNECT)
- [`import.rb`](kinnect-assessments-import/import.rb) — the loader (run in the Assessment console)
- [`templates/`](kinnect-assessments-import/templates/) — the CSVs (with a sample row each), if starting from scratch

## What gets created

| CSV                      | Target row                                              |
| ------------------------ | ------------------------------------------------------- |
| `assessments.csv`        | `ServiceItem(item_type: 'assessment', is_bundle: true)` |
| `components.csv`         | `ServiceItem(item_type: 'component')`                   |
| `variations.csv`         | `ServiceVariation` on the matching assessment           |
| `component_variants.csv` | `ComponentVariant` on the matching component            |
| `links.csv`              | `ServiceBundleItem` (assessment ↔ component)            |

**Codes are the join keys — never UUIDs.** A code typed in `links.csv` must exactly
match the `code` in `assessments.csv` / `components.csv`. The KINNECT team invents
the codes; they only need to be consistent across the four sheets.

`ServiceBundleItem` is the live table that defines an assessment's component
membership (booking, forms, serializers all read it) — so `links.csv` is what makes
an assessment actually contain its components.

## CSV column reference

### `assessments.csv`

| Column                   | Required | Notes                                                                                       |
| ------------------------ | -------- | ------------------------------------------------------------------------------------------- |
| `code`                   | ✅       | Unique across all assessments/components. The join key.                                      |
| `name`                   | ✅       | Unique per type.                                                                             |
| `description`            |          |                                                                                             |
| `booking_required`       |          | `true`/`false` (default `true`)                                                              |
| `booking_type`           |          | `requires_booking` / `no_booking_required` / `walk_in_supplier` / `internal`                |
| `appointment_duration`   |          | Minutes (default `30`)                                                                       |
| `pricing_type`           |          | `fixed` / `hourly`                                                                           |
| `service_type`           |          | `standard` / `cmwhs`                                                                         |
| `employee_eligibility`   |          | `new_employee` / `existing_employee` / `both`                                                |
| `self_bookable`          |          | `true`/`false`                                                                               |
| `self_bookable_category` |          | Required if `self_bookable=true`: `medical` / `urine` / `audiometry` / `spirometry` / `pathology` / `functional` |
| `has_variants`           |          | `true`/`false` — set `true` if it has rows in `variations.csv`                               |
| `requires_doctor_review` |          | `true`/`false`                                                                               |
| `has_outcome`            |          | `true`/`false`                                                                               |

### `components.csv`

Same columns as assessments minus `pricing_type` / `has_variants` / `has_outcome`:
`code`, `name`, `description`, `booking_required`, `booking_type`,
`appointment_duration`, `requires_doctor_review`, `self_bookable`,
`self_bookable_category`.

### `variations.csv`

| Column                 | Required | Notes                                                  |
| ---------------------- | -------- | ------------------------------------------------------ |
| `assessment_code`      | ✅       | Must match a `code` in `assessments.csv`               |
| `name`                 | ✅       | e.g. "Baseline"                                        |
| `code`                 |          | Optional — auto-generated from `name` if blank         |
| `sort_order`           |          | Integer display order                                  |
| `active`               |          | `true`/`false`                                         |
| `booking_type`         |          | Optional override (defaults from the assessment)       |
| `employee_eligibility` |          | Optional override                                      |

### `component_variants.csv`

The component-side parallel to variations (e.g. an Audiometry component with
"Standard" vs "Pre/Post Shift" variants).

| Column           | Required | Notes                                          |
| ---------------- | -------- | ---------------------------------------------- |
| `component_code` | ✅       | Must match a `code` in `components.csv`        |
| `name`           | ✅       | e.g. "Standard"                                |
| `code`           |          | Optional — auto-generated from `name` if blank |
| `sort_order`     |          | Integer display order                          |
| `active`         |          | `true`/`false`                                 |
| `description`    |          |                                                |

### `links.csv`

| Column                   | Required | Notes                                                                                  |
| ------------------------ | -------- | -------------------------------------------------------------------------------------- |
| `assessment_code`        | ✅       | Must match a `code` in `assessments.csv`                                                |
| `component_code`         | ✅       | Must match a `code` in `components.csv`                                                 |
| `position`               |          | Order of the component within the assessment (integer)                                  |
| `variation_name`         |          | Blank = link applies to all assessment variations. Set to a `variations.csv` `name` = that variation only. |
| `component_variant_name` |          | Blank = no specific component variant. Set to a `component_variants.csv` `name` (for this component) = that variant. |

**Fill order:** components → assessments → variations → component variants → links.

## Input handling

- **Booleans** accept `Yes/No`, `true/false`, `1/0`, `y/n` (case-insensitive).
- **Enums** are normalized (lower-cased, spaces/hyphens → underscores) then validated,
  so `"Requires Booking"` → `requires_booking`. An unrecognized value is reported with
  the allowed set — it does not silently fall through to a default.
- **Idempotent:** existing items (by `code`), variations (by assessment + `name`), and
  links are skipped. Re-running only adds what's missing.
- **Skip-and-report:** a bad row is reported (`file`, row number, validation errors)
  and skipped; good rows still load. The run is never aborted by one bad row.

## Generating the CSVs from staging data

To seed the four CSVs with the catalog already in staging (so KINNECT edits/appends
rather than starting blank), run the exporter. It writes the **same columns** the
importer reads, so the output round-trips straight back through `import.rb`.

```ruby
# Assessment Rails console (read-only)
ENV["OUTPUT_DIR"] = "/tmp/kinnect"
load "/tmp/kinnect/export.rb"     # or paste the whole file
```

Writes `assessments.csv`, `components.csv`, `variations.csv`,
`component_variants.csv`, `links.csv` into `OUTPUT_DIR` (overwriting on re-run).
`links.csv` resolves UUID FKs back to codes / variation names — never UUIDs.

### Bundle into one Excel workbook (optional)

To share a single file with KINNECT, merge the CSVs into a `.xlsx` — one sheet per
CSV plus a **Legend** sheet documenting the allowed values for every enum/boolean
column. Run it right after the export (caxlsx is already loaded in the console):

```ruby
ENV["OUTPUT_DIR"] = "/tmp/kinnect"
load "/tmp/kinnect/build_workbook.rb"    # -> /tmp/kinnect/kinnect_catalog.xlsx
```

Works on filled-in CSVs too, not just freshly exported ones. Output name overridable
via `ENV["WORKBOOK_NAME"]`.

## Running it (staging)

`carelever_assessment` is **single-tenant** (no Apartment) — a staging console
connects to the staging DB automatically. No tenant switching, no extra config.

> ⚠️ Writes to the **live staging catalog**. Always dry-run first.

### 1. Get the four CSVs onto the console host

Put `assessments.csv`, `components.csv`, `variations.csv`, `links.csv` (and
`import.rb`) in a folder reachable from the console, e.g. `/tmp/kinnect` — scp/paste
into the ECS container, or download from S3.

### 2. Dry run (validate only — writes nothing)

```ruby
# Assessment Rails console (RAILS_ENV=staging bin/rails console, or ECS Exec)
ENV["CSV_DIR"] = "/tmp/kinnect"
ENV["DRY_RUN"] = "true"
load "/tmp/kinnect/import.rb"    # or paste the whole file
```

Reports `created / skipped / errors` per file inside a rolled-back transaction.
Fix any unmatched codes / invalid enum values in the CSVs and re-run until clean.

### 3. Real run

```ruby
ENV["CSV_DIR"] = "/tmp/kinnect"
ENV["DRY_RUN"] = "false"
load "/tmp/kinnect/import.rb"
```

`load` re-runs the script each time. Re-running after a partial/failed run is safe —
already-created rows are skipped.

### ENV reference

| Variable  | Required | Notes                                                       |
| --------- | -------- | ----------------------------------------------------------- |
| `CSV_DIR` | ✅       | Folder holding the four CSVs (missing files are skipped)    |
| `DRY_RUN` |          | `"true"` → validate + report, roll back. Default `"false"`. |

## Verification (Assessment console)

```ruby
# Counts created by this load
ServiceItem.where(item_type: "assessment").count
ServiceItem.where(item_type: "component").count
ServiceVariation.count
ServiceBundleItem.count

# Spot-check one assessment's components
a = ServiceItem.find_by(code: "KIN-PEM")
a.bundle_components.pluck(:code, :name)
a.service_variations.pluck(:name, :code)

# Component membership incl. variation-specific links
a.service_bundle_items.pluck(:component_id, :service_variation_id, :position)
```

Checklist:

1. Assessment / component / variation / link counts match the CSV row counts.
2. Every assessment in `links.csv` resolves its components (no rows skipped for unmatched codes).
3. Variation-specific links point at the right `ServiceVariation`.
4. Re-running the import is idempotent (all rows skipped on the second run).
