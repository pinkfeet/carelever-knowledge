## Initial Migration (can re-run)

! Note that this doc includes 'Internal users' as well. However, it doesn't include Monitor referrals.
Please check [`initial-migration.md`](https://github.com/cloudhealthkinnect/carelever_assessment/blob/development/docs/data-sync/initial-migration.md) for the details.

### Export

13 files

```
script/migrate/netsuite_locations_export.rb             Organisation console
script/migrate/suburbs_export.rb                        Organisation console
script/migrate/companies_export.rb                      Organisation console
script/migrate/company_subscriptions_export.rb          Organisation console
script/migrate/company_micro_subscriptions_export.rb    Organisation console
script/migrate/suppliers_export.rb                      Organisation console (locations table)
script/migrate/sites_export.rb                          Company service console
script/migrate/positions_export.rb                      Company service console
script/migrate/contacts_export.rb                       Company service console (people table)
script/migrate/invoicing_entities_export.rb             Company service console
script/migrate/company_defaults_export.rb               Company service console
script/migrate/internal_users_export.rb                 Auth service console
script/migrate/external_users_export.rb                 Auth service console
```

For the above scripts, please extract the data with the following steps.

1. Access the related micro-service, e.g., Authentication, Organisation, or Company, using the AWS Session Manager (e.g., `dev-authentication-service`).
2. Start the Rails console: `rails console`
3. Switch to the tenant and set the environment variables (copy and paste the following code):

```ruby
Apartment::Tenant.switch!('f7290827-c426-4e92-8ba6-45fd12ca5a70')

ENV['MIGRATION_BUCKET'] = 'carelever.uploads.staging'
ENV['MIGRATION_PREFIX'] = 'migrations'
ENV['MIGRATION_ENV']    = 'staging'
```

4. Run the export script. Copy the entire content from a file and paste into the console. Each file will create separate files in S3.

5. Verify that the file has been generated in [S3](https://ap-southeast-2.console.aws.amazon.com/s3/buckets/carelever.uploads.development?prefix=migrations%2Fdevelopment%2F&region=ap-southeast-2)

### Run by chunks

```ruby
ENV['MIGRATION_LIMIT'] = '25000' # cap records this run
load "script/migrate/contacts_export.rb"

# → "Chunk full — continue with MIGRATION_AFTER_ID=<uuid>"

ENV['MIGRATION_AFTER_ID'] = '<uuid>' # paste cursor, run again
load "script/migrate/contacts_export.rb" # repeat until no "Chunk full" line
```

### Import

Before start scripts, make sure RBAC is placed.

```sh
bundle exec rake rbac:sync APPLY_ROLE_PERMISSIONS=true
```

```
script/migrate/netsuite_locations_import.rb             Assessment console
script/migrate/suburbs_import.rb                        Assessment console
script/migrate/companies_import.rb                      Assessment console
script/migrate/company_subscriptions_import.rb          Assessment console
script/migrate/company_micro_subscriptions_import.rb    Assessment console
script/migrate/sites_import.rb                          Assessment console
script/migrate/positions_import.rb                      Assessment console
script/migrate/contacts_import.rb                       Assessment console
script/migrate/invoicing_contacts_import.rb             Assessment console
script/migrate/suppliers_import.rb                      Assessment console
script/migrate/internal_users_import.rb                 Assessment console (requires suppliers)
script/migrate/external_users_import.rb                 Assessment console
script/migrate/companies_backfill_account_manager.rb    Assessment console - backfill
script/migrate/company_defaults_import.rb               Assessment console
```

Please keep the order of scripts. For each script, please set `SNAPSHOT_KEY`.

1. Access the Assessment service via the AWS Session Manager(e.g., `dev-assessment-service`).
2. Start the Rails console: `rails console`
3. Set the environment variables (replace the filename below). Load the script after changing `SNAPSHOT_KEY`.

```ruby
ENV['MIGRATION_BUCKET'] = 'carelever.uploads.staging'
ENV['SNAPSHOT_KEY'] = 'migrations/staging/internal-users/JSON-FILE-NAME.json'
load "script/migrate/internal_users_import.rb"
```
