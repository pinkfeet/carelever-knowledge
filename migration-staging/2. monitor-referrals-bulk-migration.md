# Monitor → Assessment — Referral bulk migration runbook

Operational guide for migrating **live referral data** (~50k Monitor referrals → many Assessment cycle-referrals) using the CT-4461 pipeline in `carelever_assessment/script/migrate-monitor/`.

**Canonical specs (field maps, grouping rules, billing):**

- [`carelever_assessment/script/migrate-monitor/specs/README.md`](../../carelever_assessment/script/migrate-monitor/specs/README.md)
- Manual test cases: [`TESTING-REFERRALS.md`](../../carelever_assessment/script/migrate-monitor/TESTING-REFERRALS.md)

**Related KB:**

- [Initial data migration](initial-data-migration.md) — Phase 0 org/users
- [Monitor migration plan](../monitoring/migration-plan.md) — catalog field mapping (Appendix C covers DB-to-DB for settings; **referrals use the per-file JSON pipeline below**)

---

## Overview

| Phase                   | Where                  | What                                                            |
| ----------------------- | ---------------------- | --------------------------------------------------------------- |
| **0 — Foundation**      | Assessment             | Companies, sites, users, NetSuite locations                     |
| **1 — Catalog mapping** | Assessment             | Monitoring items → service items; units → components (blocking) |
| **2 — Export**          | **Monitor** console    | One JSON file per referral → S3 or local folder                 |
| **3 — Import**          | **Assessment** console | Read JSON folder; upsert N cycle-referrals per file             |

Scale: ~50k Monitor referral files. Each file fans out to **multiple** Assessment referrals (one per program per cycle). Exports/imports are **sharded, resumable, and idempotent**.

## Prerequisites (before bulk)

### Phase 0 — already in Assessment

- Parent accounts, companies, sites, positions
- Internal users (doctors, assessors) for `created_by_id` / note authors
- NetSuite locations (billing context)

See [initial-data-migration.md](initial-data-migration.md).

### Phase 1 — catalog (blocking)

Complete before any referral import:

1. Settings export/import: [`script/migrate-monitor/settings/`](../carelever_assessment/script/migrate-monitor/settings/README.md)
2. Mapping spreadsheets: [`mapping/MAPPING.md`](../carelever_assessment/script/migrate-monitor/mapping/MAPPING.md)
   - `monitoring_item_mapping.xlsx` — test item → assessment / variation
   - `unit_component_mapping.xlsx` — unit → component / component_variant

Unmapped catalog references are **skipped with report** — never silently nulled.

### Schema — Assessment migrations applied

- `referrals.monitor_cycle_key` (string, unique) — upsert key
- `referrals.monitor_referral_id` (uuid) — reconciliation
- `services.result_data`, `services.result_config` (jsonb) — TIRR payload

### Console access

| App                 | Typical access                                      |
| ------------------- | --------------------------------------------------- |
| Monitor (export)    | Bastion → **monitoring** Rails console, or ECS Exec |
| Assessment (import) | Bastion → **assessment** Rails console, or ECS Exec |

Bastion helper: [`aws/bastion/bastion.sh`](../aws/bastion/bastion.sh) (select `monitoring` or `assessment`).

Monitor is multi-tenant (Apartment). Default dev tenant:

```ruby
Apartment::Tenant.switch!("f7290827-c426-4e92-8ba6-45fd12ca5a70")
```

Override with `ENV["TENANT_ID"]` on export.

| Env var            | Meaning                                                     |
| ------------------ | ----------------------------------------------------------- |

| `MIGRATION_PREFIX` | Optional key prefix                                         |

```

./carelever_assessment/script/migrate-monitor/referrals/fetch_bundle.sh <referral_id> <RUN_ID>
```

## Phase 2 — Export (Monitor console)

Script: [`referrals_export.rb`](../../carelever_assessment/script/migrate-monitor/referrals/referrals_export.rb)

**How to run:** set `ENV` in the **Ruby** console, then `load` the script. Do not mix shell env prefixes with `load`.

On ECS/bastion you can paste the entire file, or:

```ruby
load "script/migrate-monitor/referrals/referrals_export.rb"
```

### Full bulk to S3

```ruby
ENV["MIGRATION_BUCKET"] = "carelever.uploads.staging"   # or prod bucket
ENV["TENANT_ID"]        = "f7290827-c426-4e92-8ba6-45fd12ca5a70"
ENV["RUN_ID"]           = "2026-06-23-bulk-v1"          # fixed — enables resume
ENV["SKIP_EXISTING"]    = "true"                        # default — skip keys that exist
ENV["BATCH_SIZE"]       = "500"                         # default — memory / N+1 control
load "script/migrate-monitor/referrals/referrals_export.rb"
```

When finished, note the `RUN_ID` and check `_summary.json` for written / skipped / error counts.

### Parallel export (shard by date range)

Run **multiple workers** (separate consoles/tasks) with the **same `RUN_ID`** and disjoint date windows:

```ruby
# Worker 1
ENV["RUN_ID"]           = "2026-06-23-bulk-v1"
ENV["CREATED_AFTER"]    = "2020-01-01"
ENV["CREATED_BEFORE"]   = "2022-01-01"
ENV["MIGRATION_BUCKET"] = "carelever.uploads.staging"
load "script/migrate-monitor/referrals/referrals_export.rb"

# Worker 2 — same RUN_ID, next range
ENV["CREATED_AFTER"]    = "2022-01-01"
ENV["CREATED_BEFORE"]   = "2024-01-01"
# ...
```

`SKIP_EXISTING=true` makes re-runs safe: only gaps are written.

### Count referrals by created date (plan shards)

Run in the **Monitor** console (same tenant as export). Filters match `referrals_export.rb` `base_scope`: `referrals.created_at` and demo companies excluded when `SKIP_DEMO=true` (default).

```ruby
Apartment::Tenant.switch!("f7290827-c426-4e92-8ba6-45fd12ca5a70")
```

**One export shard** — same logic as `CREATED_AFTER` / `CREATED_BEFORE`:

```ruby
after  = Time.parse("2020-01-01")   # CREATED_AFTER — inclusive
before = Time.parse("2022-01-01")   # CREATED_BEFORE — exclusive

scope = Referral.all
scope = scope.where(created_at: after..)
scope = scope.where(created_at: ...before)
scope = scope.where(person_id: Person.where.not(company_id: Company.where(for_demo: true)))

puts scope.count
```

`CREATED_BEFORE` is **exclusive** (`...before`), so `2020-01-01` → `2022-01-01` means `[2020-01-01, 2022-01-01)`.

**Total migratable referrals** (no date filter, demo excluded):

```ruby
Referral.where(
  person_id: Person.where.not(company_id: Company.where(for_demo: true))
).count
```

Include demo companies:

```ruby
Referral.count
```

**Breakdown by year** (size export workers):

```ruby
Referral
  .where(person_id: Person.where.not(company_id: Company.where(for_demo: true)))
  .group(Arel.sql("DATE_TRUNC('year', created_at)"))
  .order(Arel.sql("DATE_TRUNC('year', created_at)"))
  .count
```

**Breakdown by month** (within one year):

```ruby
Referral
  .where(created_at: Time.parse("2023-01-01")..Time.parse("2024-01-01"))
  .where(person_id: Person.where.not(company_id: Company.where(for_demo: true)))
  .group(Arel.sql("DATE_TRUNC('month', created_at)"))
  .order(Arel.sql("DATE_TRUNC('month', created_at)"))
  .count
```

**Date range bounds:**

```ruby
Referral.minimum(:created_at)
Referral.maximum(:created_at)
```

**Assessment (already imported)** — `created_at` is the import stamp, not Monitor creation date:

```ruby
Referral.where.not(monitor_referral_id: nil).count
```

### Subset / smoke export

```ruby
ENV["SNAPSHOT_DIR"] = "/tmp/monitor-migrate"
ENV["REFERRAL_IDS"] = "uuid1,uuid2"
load "script/migrate-monitor/referrals/referrals_export.rb"
```

### Delta export (catch-up)

Re-export referrals whose row **or child rows** changed since a watermark:

```ruby
ENV["UPDATED_SINCE"] = "2026-06-01T00:00:00Z"
ENV["RUN_ID"]        = "2026-06-23-delta-1"   # or same RUN_ID for in-place refresh
load "script/migrate-monitor/referrals/referrals_export.rb"
```

**Scoping precedence:** `REFERRAL_IDS` > `UPDATED_SINCE` > `CREATED_AFTER`/`CREATED_BEFORE` > all.

**Note:** `UPDATED_SINCE` does not watch legacy `Evaluation`/`Result` tables — see [legacy-results-migration.md](../../carelever_assessment/script/migrate-monitor/specs/legacy-results-migration.md).

## Phase 3 — Import (Assessment console)

Script: [`referrals_import.rb`](../../carelever_assessment/script/migrate-monitor/referrals/referrals_import.rb)

Requires `ENV["SYSTEM_USER_ID"]` — a valid Assessment `User` id for note/activity authors when Monitor `action_by` cannot be resolved.

### Dry run (recommended before bulk)

Builds everything, rolls back each file's transaction, reports only:

```ruby
ENV["MIGRATION_BUCKET"] = "carelever.uploads.staging"
ENV["RUN_ID"]           = "2026-06-23-bulk-v1"
ENV["SYSTEM_USER_ID"]   = "<internal-user-uuid>"
ENV["DRY_RUN"]          = "true"
load "script/migrate-monitor/referrals/referrals_import.rb"
```

### Full bulk import

```ruby
ENV["MIGRATION_BUCKET"] = "carelever.uploads.staging"
ENV["RUN_ID"]           = "2026-06-23-bulk-v1"
ENV["SYSTEM_USER_ID"]   = "<internal-user-uuid>"
load "script/migrate-monitor/referrals/referrals_import.rb"
```

### Local folder import

```ruby
ENV["SNAPSHOT_DIR"]   = "/tmp/monitor-migrate"
ENV["RUN_ID"]         = "<timestamp-from-export>"
ENV["SYSTEM_USER_ID"] = "<internal-user-uuid>"
load "script/migrate-monitor/referrals/referrals_import.rb"
```

### Import behaviour

- **Folder-level:** processes every `<id>.json` under the `RUN_ID` prefix
- **Idempotent:** upserts by `referrals.monitor_cycle_key`
- **One DB transaction per file** — all-or-nothing per Monitor referral
- **Ledger:** `monitor_import_log` tracks `pending` / `in_progress` / `done` / `failed`
- **Failures:** logged per file; batch continues (does not abort)

### Resume after partial run

Re-run the **same** import command with the same `RUN_ID`:

- Files marked `done` are skipped
- `failed` / incomplete files are retried

```ruby
# Inspect failures
ReferralsImport::Ledger.where(status: "failed").pluck(:s3_key, :error, :log)
```

### Debug — single file

```ruby
ENV["ONLY_KEYS"] = "e5e369f1-6910-4007-a3b8-cbf3a05ea104.json"
load "script/migrate-monitor/referrals/referrals_import.rb"
```

### Reset and re-import a file

```ruby
ENV["RESET"]          = "true"
ENV["ONLY_KEYS"]      = "<referral_id>.json"
ENV["SYSTEM_USER_ID"] = "<internal-user-uuid>"
load "script/migrate-monitor/referrals/referrals_import.rb"
# then import again without RESET
```

Multi-file reset requires `ENV["RESET_CONFIRM"] = "true"`.

### Parallel import

The console driver is **sequential**. For faster bulk today:

- Run multiple Assessment consoles with disjoint `ONLY_KEYS`, or
- Split export into sub-prefixes and import each with `IMPORT_PREFIX`

Sidekiq fan-out (one job per file) is described in the spec but not wired in the script yet.

## Recommended cutover sequence

1. **Bulk load** — sharded export → import (fixed `RUN_ID`)
2. **Periodic deltas** — `UPDATED_SINCE` export → import (same or new `RUN_ID`)
3. **Freeze Monitor writes**
4. **Final delta**
5. **Switch** portals to Assessment (no ongoing reverse sync)

Pre-TIRR pure legacy results (no TIRR row) are **not migrated** — see [legacy-results-migration.md](../../carelever_assessment/script/migrate-monitor/specs/legacy-results-migration.md).

## Verification

### Minimum manual suite

Eight referrals covering in-progress, completed, multi-cycle, notes, comms, pre-TIRR, attachments — see [TESTING-REFERRALS.md § Minimum test set](../../carelever_assessment/script/migrate-monitor/TESTING-REFERRALS.md#minimum-test-set-8-referrals).

### Post-import (Assessment console)

```ruby
monitor_id = "<monitor_referral_uuid>"
cycles = Referral.where(monitor_referral_id: monitor_id)
puts "cycles: #{cycles.count}"
cycles.pluck(:monitor_cycle_key, :reference_number, :processing_mode, :referral_completed_at)

rid = cycles.first.id
puts "notes: #{ReferralNote.where(referral_id: rid).count}"
puts "comms: #{NotificationLog.where(referral_id: rid).count}"

cycles.each do |r|
  puts "#{r.reference_number}: #{r.services.pluck(:billing_status).tally}"
end
```

Checklist:

1. Cycle count matches expected fan-out
2. All `monitor_cycle_key` present; re-import is idempotent
3. Exactly one cycle has verbatim Monitor `reference_id`
4. Completed historical services → `billing_status: billed`; in-progress → `unbilled`
5. History and document counts match export JSON
6. UI: History, Results, Documents tabs

## Re-trial

All steps below run in the **Assessment** Rails console (`carelever_assessment`), **not** Monitor or reimagined. Loading `referrals_import.rb` defines `ReferralsImport` and **auto-runs** import (or reset when `RESET=true`) — set `ENV` first, then `load`.

Ledger table: `monitor_import_log` (primary key = full S3 key).

| Ledger status | Re-run same import | Data in Assessment |
| ------------- | ------------------ | ------------------ |
| `failed`      | **Retried automatically** — only `done` is skipped | Usually none (transaction rolled back on `SkipFile` / error) |
| `done`        | Skipped            | Cycle-referrals exist — use `RESET=true` to wipe and re-import |
| `in_progress` | Retried (e.g. worker crashed mid-file) | May be partial — prefer `RESET=true` if unsure |

### 1. Inspect failures

```ruby
# After at least one import run — ReferralsImport::Ledger exists only after load.
# Safer without loading the import script:
key = "migrations-monitor/referrals/test/f18cf065-9274-46de-90e3-857fe39a4d08.json"

ActiveRecord::Base.connection.select_one(
  ActiveRecord::Base.sanitize_sql_array([
    "SELECT s3_key, status, error, attempts, updated_at, LEFT(log, 500) AS log_preview FROM monitor_import_log WHERE s3_key = ?",
    key
  ])
)

# All failed keys for a run (prefix match)
ActiveRecord::Base.connection.select_all(
  "SELECT s3_key, error FROM monitor_import_log WHERE status = 'failed' AND s3_key LIKE '%/referrals/test/%' ORDER BY updated_at DESC LIMIT 20"
)
```

Or after `load "script/migrate-monitor/referrals/referrals_import.rb"` in a prior session:

```ruby
ReferralsImport::Ledger.where(status: "failed").pluck(:s3_key, :error, :log)
```

### 2. Fix root cause, then re-run (most common — e.g. catalog mapping)

**SkipFile / unmapped catalog:** no imported rows to destroy. Update `catalog_mapping.json` on the **import host** (not S3), verify, then re-import.

```ruby
path = Rails.root.join("script/migrate-monitor/referrals/catalog_mapping.json")
catalog = JSON.parse(File.read(path))
catalog["test_items"].key?("83d67cef-d564-4a50-ac1e-636797199feb")  # => true when fixed
catalog["test_items"].size
File.mtime(path)
```

Re-run — failed files are picked up automatically; **do not** set `RESET`:

```ruby
ENV.delete("RESET")
ENV.delete("RESET_CONFIRM")
ENV.delete("DRY_RUN")

ENV["MIGRATION_BUCKET"] = "carelever.uploads.staging"
ENV["RUN_ID"]           = "test"                    # same RUN_ID as the failed run
ENV["SYSTEM_USER_ID"]   = "<internal-user-uuid>"
ENV["ONLY_KEYS"]        = "f18cf065-9274-46de-90e3-857fe39a4d08.json"  # optional — single file

load "script/migrate-monitor/referrals/referrals_import.rb"
```

`ONLY_KEYS` accepts the filename alone or the full key path.

### 3. Reset ledger only (optional — usually unnecessary for `failed`)

Failed rows are retried on the next import without clearing the ledger. To force a clean ledger row before retry:

```ruby
key = "migrations-monitor/referrals/test/f18cf065-9274-46de-90e3-857fe39a4d08.json"
ActiveRecord::Base.connection.execute(
  ActiveRecord::Base.sanitize_sql_array(["DELETE FROM monitor_import_log WHERE s3_key = ?", key])
)
```

Then re-import as in §2.

### 4. Full reset + re-import (file previously `done`, or bad data landed)

`RESET=true` **hard-destroys** all cycle-referrals for that Monitor referral and **deletes** the ledger row. Required before re-import when status is `done`.

**Single file:**

```ruby
ENV["RESET"]          = "true"
ENV["MIGRATION_BUCKET"] = "carelever.uploads.staging"
ENV["RUN_ID"]         = "test"
ENV["SYSTEM_USER_ID"] = "<internal-user-uuid>"
ENV["ONLY_KEYS"]      = "f18cf065-9274-46de-90e3-857fe39a4d08.json"

load "script/migrate-monitor/referrals/referrals_import.rb"
```

**Re-import** (unset reset):

```ruby
ENV.delete("RESET")
ENV.delete("RESET_CONFIRM")

ENV["MIGRATION_BUCKET"] = "carelever.uploads.staging"
ENV["RUN_ID"]           = "test"
ENV["SYSTEM_USER_ID"]   = "<internal-user-uuid>"
ENV["ONLY_KEYS"]        = "f18cf065-9274-46de-90e3-857fe39a4d08.json"

load "script/migrate-monitor/referrals/referrals_import.rb"
```

**Multiple files** in one reset require `ENV["RESET_CONFIRM"] = "true"` (safety guard). Prefer resetting one file at a time via `ONLY_KEYS`.

### 5. Retry all failed files in a run

Fix the shared root cause (mapping, schema, etc.), then re-run the **whole** import with the same `RUN_ID` — no `ONLY_KEYS`. Every `failed` / non-`done` key is processed; `done` keys are skipped.

```ruby
ENV.delete("RESET")
ENV.delete("ONLY_KEYS")
ENV.delete("DRY_RUN")

ENV["MIGRATION_BUCKET"] = "carelever.uploads.staging"
ENV["RUN_ID"]           = "2026-06-23-bulk-v1"
ENV["SYSTEM_USER_ID"]   = "<internal-user-uuid>"

load "script/migrate-monitor/referrals/referrals_import.rb"
```

### 6. Dry-run before re-trial

```ruby
ENV["DRY_RUN"] = "true"
# … same RUN_ID / ONLY_KEYS as the retrial …
load "script/migrate-monitor/referrals/referrals_import.rb"
ENV.delete("DRY_RUN")
```

### Quick decision guide

| Situation | Action |
| --------- | ------ |
| `SkipFile` / unmapped catalog, status `failed` | Fix `catalog_mapping.json` on host → re-run (§2). No `RESET`. |
| Import succeeded but wrong mapping | `RESET=true` + `ONLY_KEYS` → re-import (§4). |
| Crash mid-file, status `in_progress` | Re-run; if duplicate/partial data, `RESET` then re-import. |
| Status `done`, need to re-import | Must `RESET` first — otherwise skipped. |

## Troubleshooting

| Symptom                         | Action                                                                     |
| ------------------------------- | -------------------------------------------------------------------------- |
| Export OOM / slow               | Lower `BATCH_SIZE`; shard date ranges across workers                       |
| Import `UnmappedCatalog` / skip | Fix Phase 1 mapping; re-export affected referrals                          |
| Partial import after crash      | Re-run same `RUN_ID` import — ledger skips `done`                          |
| Wrong data on one referral      | `RESET=true` + `ONLY_KEYS` + re-import                                     |
| Failed keys pile up             | `ReferralsImport::Ledger.where(status: 'failed')` — fix root cause, re-run |
| Company missing in Assessment   | Referral skipped at export (`SKIP_DEMO` / missing company)                 |

Export summary: `s3://<bucket>/migrations-monitor/referrals/<RUN_ID>/_summary.json`
